搭建博客的过程

在寒假期间用Hexo搭建了一个自己的博客，来说说我搭建博客的过程吧
具体操作如下：
1.先注册好一个GitHub账号，后面要把我们的博客发布到github pages
2.然后在电脑上安装好Git，没有git哪来hub：）
3.继续在电脑上安装NodeJS，因为Hexo是基于NodeJS编写的，所以需要安装NodeJS和npm工具
4.后面就可以开始整活了，先在GitHub上创建一个新的代码仓库用于保存项目文件，点击Your repositories，进入仓库页面，然后点击New按钮，进入仓库创建页面，填写仓库名，格式必须为<用户名>.github.io（不然到时浏览器连不上去），然后点击Create repository就完成了第一步
5.Then我们安装Hexo
在gitbash中输入：
npm install -g hexo-cli
让后创建一个项目hexo-blog并初始化：
hexo init hexo-blog
cd hexo-blog
npm install
6.我们可以使用Fluid来更换主题
如安装主题：
下载最新的release版本解压到themes目录，并将解压出的文件夹重命名为fluid
如指定主题：
修改Hexo博客目录中的_config.yml：
theme: fluid
language: zh-CN
7.为什么不创建创建「关于页」呢：
hexo new page about
创建成功后，编辑博客目录下/source/about/index.md，添加 layout 属性，如下：
title: about
date: 2025-02-10 13:03:47
layout: about
这里就写关于页的正文，支持 Markdown, HTML哦
8.创建文章啦
修改Hexo博客目录中的 _config.yml，打开这个配置是为了在生成文章的时候生成一个同名的资源目录用于存放图片文件：
post_asset_folder: true
执行如下命令创建一篇新文章：
hexo new post XXXX
执行完成后在source_posts目录下生成了一个md文件和一个同名的资源目录(用于存放图片)哈
在XXXX.md中添加如下模板:
title: XXXX
date: XXXX
tags:
- XXXX
categories:
- XXXX
其中的内容可以根据自身需求进行更改
9.来一点个性化页面展示
页面的标题等位置显示的是默认文字，可以改下显示一些个性化的信息
a.浏览器tab页名称
修改根目录下_config.yml中的title字段
b.博客标题
主题目录themes\fluid下_config.yml中的blog_title字段
c.主页正中间的文字
主题目录themes\fluid下_config.yml中的text字段
10.后面添加阅读量统计
其实Fluid主题写好了统计阅读量的代码，但是缺少相应配置所以没有开启，所以需要借助三方服务来统计阅读量，可以用 Leancloud的免费服务来进行统计

12.发布到GitHub Pages
要安装hexo-deployer-git哈
在git bash中输入：

然后修改根目录下的_config.yml，配置GitHub相关信息

repo: https://github.com/cpython333/cpython333.github.io.git
